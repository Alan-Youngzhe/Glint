Pedagogical Model · 三层教学
理解是分层的，
也是可以来回跳的。
L3
架构层
技术栈、框架选型、整体设计 — ⌥4 架构 Treemap + Tech Stack 面板
L2
模块层
模块如何组织、谁调用谁、业务逻辑怎么走 — ⌥2 调用关系 · ⌥3 执行路径
L1
代码层
逐行逻辑、为什么这么写、数据流 — ⌥1 就近浮卡
对应论文发现：专业审计员遵循「全局→局部→细节」理解流，并在各层间反复跳转修正心智模型。⌥ 的四个键让这种跳转即时发生，并留下可追溯的轨迹。
